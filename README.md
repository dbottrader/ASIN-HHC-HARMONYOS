## What This Repository Is

ASIN-HHC HarmonyOS is a convergence archive. It preserves creative-symbolic work, UI experiments, geometry/glyph research, agentic critique, and reproducible technical artifacts under one structured repository.

The repo is intentionally evidence-aware:

- symbolic artifacts stay symbolic unless verified;
- local tools are marked as local until independently reviewed;
- witness reports are not treated as reproduction;
- reproducible claims require source inputs, scripts, commands, and expected outputs;
- production claims require deployment and operational hardening.

## ASIN Format

ASIN means:

```text
Anchor     — reference point or source object
Shape      — geometry, structure, visible form
Intention  — interpreted or assigned function/purpose
Number     — count, ratio, frequency, timestamp, binary, or numeric encoding
```

---

## Evidence Ladder

```text
LOOM -> LOCAL -> WITNESS -> REPRO -> PRODUCTION
```

| Class | Meaning |
|---|---|
| LOOM | Symbolic, mythic, visual, poetic, interpretive |
| LOCAL | Runnable or inspectable locally |
| WITNESS | Reviewed or critiqued by another agent/person/process |
| REPRO | Reproducible from source inputs and commands |
| PRODUCTION | Deployed, monitored, maintained, and operationally hardened |

---

## Repository Intent

The goal is not to inflate claims. The goal is to preserve the whole project with clear boundaries so each artifact can be developed, reviewed, reproduced, or promoted correctly.

This is the working rule:

> Preserve the vision. Protect the evidence boundary. Let the repository become the lineage.
