# CP8 Ecosystem Map

This document maps the important working structure of the ASIN-HHC / CP8 / HarmonyOS project as of the July 2026 import branch.

---

## 1. System Purpose

CP8 is the convergence layer for the ASIN-HHC HarmonyOS ecosystem. It organizes symbolic design, geometric intelligence, provenance, agent review, UI artifacts, hashing, and reproducible handoff packages into a coherent archive.

The core aim is not to collapse everything into one claim. The aim is to keep each artifact in its proper lane while preserving the relationships between them.

Examples:

Evidence boundary: `REPRO` only when procedures are executable and deterministic.

### 2.5 Agent/Witness Layer

Examples:

- CP8 convergent critique.
- Weaver Epistemic Promotion Gate.
- Cathedral spine MVP0 witness handoff.
- Shock Kernel template-only package.

Evidence boundary: `WITNESS` or `REPRO` depending on the level of independent verification.

## 3. Core Technical Concepts

### ASIN

ASIN means:

- **Anchor** — the reference point or grounding object.
- **Shape** — the geometry, structure, or visible form.
- **Intention** — the interpreted or assigned function/purpose.
- **Number** — count, ratio, frequency, timestamp, binary, or numeric encoding.

### HHC

HHC is the human/codex/collaboration context. It bridges lived experience, creative design, interface building, and symbolic system-making.

### CP8

CP8 is the convergence protocol. It organizes symbolic, technical, geometric, and provenance layers into a reproducible archive where possible.

### HarmonyOS Mapping

HarmonyOS is used as an architectural metaphor:

- Super Device -> CP8 Lattice.
- Soft Bus -> Git, issues, inbox, repo links, and message flow.
- Microkernel -> minimal trusted core.
- Manifest topology -> explicit records and deterministic paths.
- Verification -> hashing, Merkle records, tests, and audit trails.

## 6. Promotion Rule

Every artifact should answer four questions:

1. What is it?
2. What evidence class does it belong to?
3. What data or files support it?
4. What would be required to promote it to a stronger evidence class?

This prevents symbolic material from being mistaken for verified engineering, while still preserving the symbolic layer as part of the living archive.
