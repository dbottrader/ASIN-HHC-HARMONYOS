## Core Identity

**ASIN-HHC HarmonyOS** is a symbolic-operational framework combining:

- ASIN format: Anchor, Shape, Intention, Number.
- HHC: Happy Hippie Codex / Harmonic Human-Collaboration context.
- CP8: convergence-oriented project layer for geometry, provenance, resonance, agents, and evidence tracking.
- HarmonyOS mapping: a distributed coordination metaphor using device-lattice, soft-bus, microkernel, manifest topology, and deterministic verification.

The project treats symbolic glyphs, geometric forms, user interfaces, provenance records, agent outputs, and reproducible technical artifacts as separate layers of one living archive.

## Evidence Boundary

This repository can hold mythic, symbolic, experimental, engineering, and reproducibility artifacts together, but each artifact should declare its evidence class.

Recommended evidence classes:

- `LOOM` — symbolic, narrative, mythic, interpretive, visual orientation.
- `LOCAL` — locally reproduced or locally executable.
- `WITNESS` — independently reviewed or cross-agent verified.
- `REPRO` — deterministic reproduction available with tests/scripts/data.
- `PRODUCTION` — deployed, maintained, monitored, and operationally hardened.

No artifact should claim scientific, clinical, hardware, security, financial, or production authority unless the supporting evidence is present in the repository.
