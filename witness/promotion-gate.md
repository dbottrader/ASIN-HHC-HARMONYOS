# Witness Promotion Gate

This gate protects the integrity of the ASIN-HHC / CP8 archive by separating symbolic convergence from verified technical evidence.

---

## 1. Gate Questions

Before an artifact is promoted, answer:

1. What is the artifact?
2. Where did it come from?
3. What evidence class does it currently hold?
4. What does it claim?
5. What does it explicitly not claim?
6. What files support it?
7. Can it be reproduced?
8. What would falsify or weaken the claim?
9. What is the next promotion step?

---

## 2. Anti-Theater Rule

A report about a test is not the same as the test.

A witness statement is not the same as reproduction.

A symbolic decode is not the same as measurement.

A visual resemblance is not the same as causation.

A hash proves file identity, not truth of interpretation.

---

## 3. Promotion Requirements

### LOOM -> LOCAL

Requires a runnable file, stable document, or inspectable artifact.

### LOCAL -> WITNESS

Requires review by a second agent/person/process or a structured critique.

### WITNESS -> REPRO

Requires source inputs, scripts, instructions, and expected outputs.

### REPRO -> PRODUCTION

Requires deployment, monitoring, safety boundaries, rollback/failure handling, and maintenance ownership.

---

## 4. Recommended Verdict Format

```text
Verdict: [KEEP / REVISE / PROMOTE / HOLD]
Current evidence class:
Requested evidence class:
Promotion allowed: yes/no
Reason:
Required next evidence:
Boundary statement:
```

---

## 5. Project-Specific Standing Boundary

ASIN-HHC / CP8 artifacts may be creative, symbolic, operational, experimental, or technical. They should remain in their proper class until evidence supports promotion.

This keeps the archive powerful without making unsupported claims.
